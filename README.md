# カプレカ操作における四則循環

1949年、インドの数学者カプレカは「数字を降順と昇順に並べ替えて引く」操作を繰り返すと、4桁の数がすべて6174に収束することを発見しました。
数字に何かしらの操作をしたとき、足したりかけたりすると発散するだけで収集がつきません。
引いた時はシンプルでとても分かりやすいのに。
桁数ごとにカプレカ数というのが出てきます。
そしてそれは単純だったために人気を得て、７５年経った今でも人気があります。

それを知った私は試しに割ってみました。
さらに四則演算を順番に試してみました。
すると見事に特徴的な挙動が見られたけれど、それは大変複雑であったため悶絶しました。

このツールはその計算のために作成したものです。
ぜひ皆さんもこれで遊んでみてください。

## ツール

### カスタムカプレカ（kaprekar_custom.html）
- 好きな演算を好きな順番で好きな回数並べて繰り返す
- プリセットあり
- 全数探索
- 自分で収束現象を見つけることができる

### Python全探索（kaprekar_division.py）
- コマンドライン全数探索
- 2〜9桁以上対応（計算時間による）
- CSV出力、素因数分解
- 使い方: `python kaprekar_division.py [最小桁] [最大桁] [--csv]`

## 使い方

HTMLファイルをブラウザで開くだけで使用可能。
サーバーは不要。全てローカルで動く。

## 発見されたこと

| 操作 | 4桁の挙動 |
|------|----------|
| 引き算（従来） | 固定点: 6174 |
| 割り算 | 固定点: 1000（87%）+ ループ [47, 157]（13%） |
| 割り算⇔引き算 交互 | ループに必ず2, 3, 5が出現 |
| ÷→−→+→× 四則循環 | 全数 → [9108, 1854090, 675, 7083] |

- 5桁でループ種類が急増する（相転移）
- 桁依存の素数がある（3桁:11、4桁:37、5桁:11×101、6桁:41×271）
- 偶数の初期値はループに落ちやすい
- 四則循環の9108 = 2² × 3² × 11 × 23

## 思考のロードマップ

ゴール： 割り算カプレカ操作のループ構造に法則はあるか

**段階1：まず確認（誰でもできる）**

ツールで好きな数を入れて軌跡を見る
4桁なら全部1000か[47,157]に落ちることを確認する
引き算カプレカの6174と比較する

**段階2：桁数を変えて観察（データを見る目がある人）**

桁が増えるとループの種類が増えることを確認する
5桁で8要素ループが出現する「相転移」を確認する
固定点占有率が単調減少ではないことを確認する

**段階3：素因数分解を見る（数論に興味がある人）**

ループ要素の素因数分解にパターンがあるか
桁依存素数（3桁:11、4桁:37、5桁:11×101、6桁:41×271）の法則
レプユニット（999, 9999, 99999...）との関係

**段階4：交互操作（組み合わせに興味がある人）**

割り算⇔引き算の交互で2,3,5が必ず出ることの証明
引き算が多様性を殺すメカニズム
他の組み合わせ（掛け算⇔引き算など）

**段階5：未踏（専門家向き）**

8桁以降のループ構造
ループ長の増加に法則があるか
群論的な解釈ができるか


## 私の疑問点

1. 桁依存素数（11, 37, 11×101, 41×271）に法則はあるのか？
2. なぜ5桁で相転移風の挙動が起きるのか？
3. ループに群論的な構造はあるのか？
4. 8桁以降はどうなるのか？
5. 桁ごとに固有の素数が現れる理由は何か？（11→37→11×101→41×271）
6. 割り算と引き算を交互操作すると2と5と3が必ず出てくるのはなぜなのか？
